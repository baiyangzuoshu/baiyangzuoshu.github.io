《龙光诗经》：旧式蒙学阅读里的一部《诗经》

  《诗经》向来是熟书。熟到后来人提起它，往往先想到的是“经学”“古典”“名篇”，反而忘了它也曾经是私塾案头、书肆柜中极为常见的一类书。真见到这种旧式小本，感觉便不一样了。纸色旧，边角磨损，封签歪斜，甚至连穿线和版心都带着很重的旧日气息。它不是为陈列而生的珍本，更像是一册确实被翻读过、被讲解过、被拿来启蒙过的书。

  从封面题签看，这册书直接题作《龙光诗经》。就气息而言，它更像旧式坊刻流通本，而不是供书斋珍藏的高古善本。可也正因为如此，它反倒更有意味。许多真正进入民间阅读生活的书，并不是靠少数孤本流传，而是靠这种一印再印、装帧朴素、价廉而耐翻的小册子，慢慢把经典送到一代又一代读书人的手边。

  《诗经》本身是中国最早的一部诗歌总集，三百篇流传既久，后来又成为经学核心。可若只从“经”的角度去看，往往容易把它看得过于庄严。翻到这种旧本时，反而会觉得它首先是一部“可以朗读、可以讲说、可以拿来教人识字与明义”的书。书页上的大字、小注、圈点和分栏，都让人想到旧式课堂里的讲读场景：先生领读，学童跟诵，一边记字，一边识义。

  这也是此类小本最可观处。它并不追求注疏浩繁，而更偏向于“够用”的编排方式。正文醒目，旁注直白，版面安排尽量让初学者一眼能分清经文与解释。这种做法当然谈不上精深考据，却很贴近日常阅读。经典若真要进入人的记忆，最初往往并不是以学术专书的形式，而正是以这种更易上手、更便于诵习的面貌。

  从内页看，书中版心、行格与字样都保存着晚近石印或翻刻本常见的秩序感。页面虽然已有磨损，墨色也并非处处匀净，但仍能看出旧式排版的讲究。这样的版式，与今天整洁的横排简体印本完全是两套阅读经验。现代读者习惯了快速浏览，旧本却天然要求人慢下来：字要逐行认，句要逐层断，连翻页的动作都像是阅读的一部分。

  《诗经》的内容本来就带着极强的口耳传统色彩。国风可歌，雅颂可诵，许多篇章之所以历久不衰，也正因为它们本来就是要进入声音中的。旧式《诗经》小本的价值，也恰在这里。它不是只把经典作为供人膜拜的对象，而是仍旧维持着一种“拿来读、拿来教、拿来背”的实用性。经典一旦仍保有这层实用性，便还活在人的日常里。

  所以像《龙光诗经》这样的书，未必罕见，却很能见时代风气。它让人看到，经典如何在近代以前和近代之初的教育世界中继续流通，也让人想到《诗经》并不只是文人书房里的高头讲章，更是曾进入寻常塾课、进入启蒙秩序的一册读本。其意义未必在版本多么名贵，而在它确实承载过那种已经逐渐远去的阅读生活。

  从藏书的眼光看，我向来喜欢这种“不那么显赫”的书。它们不一定以稀见取胜，却常常更有生活气。封衣、题签、纸色、磨痕，甚至页边的残损，都在告诉人：这册书不是后来人为古董而留，而是真正被人使用过。经典之为经典，不只因为被收入目录和史册，也因为它曾经一遍遍被读、被讲、被抄、被背。像《龙光诗经》这样的旧本，正把这种阅读世界的余温留了下来。

《古文观止》：一部人人读过的小书，换作旧本来看便另有意味

  《古文观止》原本是人人都熟的书，熟到几乎不必多说。小时候读古文，许多人案头总绕不过它；后来读书稍多，又常嫌它“太熟”“太通行”，仿佛因为流传太广，便减了几分珍贵。可真见到旧本，心里往往还是会微微一动。书一旦从现代排印本回到旧式线装的形态，那种熟极了的内容，忽然便又生出些新鲜意味。

  这部《古文观止》，从签条看，当是“民国癸亥年春，上海记书庄新刊”。未必是什么稀见善本，却自有一种旧时坊刻流传本的气象。纸色已渐暗，封衣磨损，边角多有残痕，但版框尚整，字口也还分明。这样的书，拿在手里时最能让人意识到，所谓“古文”，原来并不只是课本中被反复讲解的篇章，它也曾经是旧时代读书人随手翻检、日日诵习的实物。

  这组书影拍得并不连贯，倒像是随手翻到哪里便留一页。前面所见，仍多是《左传》《战国策》中的名篇，像《郑伯克段于鄢》《秦晋殽之战》《苏秦以连横说秦》《司马错论伐蜀》之类，篇名一出，古文的筋骨便立刻显出来了。先秦文字最见气势，也最见叙事与辞令之妙；《古文观止》以这些篇章开路，原就是要让读书人先见古文之源头。

  再往后翻，便到《史记》诸赞。《五帝本纪赞》《项羽本纪赞》《孔子世家赞》这些文字，篇幅不长，却有太史公笔力凝聚后的沉着气。它们不像长篇传记那样铺陈展开，而是在极短的篇幅里，把人物、史识与感慨一并收住。这样编排，也正见《古文观止》的选文眼光：并不单求篇幅宏大，而更看重文章之可诵、可观、可为法。

  到了卷六以后，书里那种“古文教材”的面目便愈发明显起来。《前出师表》《陈情表》《师说》这样几篇，几乎是每个稍习古文者都绕不过的文字。诸葛亮写得忠恳，李密写得沉至，韩愈则直截有力，三家笔法不同，而都足以立言。今日重看，也许未必句句新鲜，但放在旧本中读，却仍能感到这些文章之所以流传数百年的缘故：它们并不是因“应试”才被记住，而是本来就有能叫人一读不忘的力量。

  此后又见《驳复雠议》《梅圣俞诗集序》《上梅直讲书》《送天台陈庭学序》诸篇，从柳宗元到欧阳修、苏轼，再到宋濂，一路看下来，唐宋以来古文的脉络便大致具在其中了。论说、书信、序文、表章，各有体制，各有神情。《古文观止》最见功夫处，也许正不在“搜罗最全”，而在“去取得当”。它替后来读书人拣出了最容易见骨力、见气脉、见法度的一批文章，于是成了一部流传最广的古文选本。

  也正因为它太通行，后人有时反倒容易轻看它。似乎人人都读过，便不值得郑重对待。其实恰恰相反。像《古文观止》这样的书，未必罕见，却最能说明古文传统究竟是如何进入日常阅读生活的。许多真正影响深远的书，并不是靠秘本珍椠流布后世，而是靠这种一印再印、家塾案头常备、童蒙与士子都能接触的小部头，慢慢渗进无数人的记忆里。

  从藏书的眼光看，这部民国新刊本《古文观止》最可喜处，也许不在版本多么高古，而在它把“读过的书”重新变成了“可以细看的一本书”。书上的污痕、旧封、字迹、残破，连同每一页版式，都在提醒人：古文并不只存在于今天整洁的排印文本里，它也曾经以这种旧式面貌，被人翻过、读过、背过、讲过。经典之所以成为经典，往往并不只因为文章本身伟大，也因为它曾在漫长岁月里，真实地进入过一代代人的日常。

  所以这样的书，放在架上时也许不算最惹眼；但真抽出来翻阅，却总有绵长余味。它让人想到，所谓古文传统，并不是一个抽象的文学名目，而是一册册被人用过的旧书，一篇篇被人反复诵习的文章。藏书之乐，有时也正在这里：不是只看稀见与珍贵，而是在一部人人都知道的书里，重新看见旧日阅读世界的温度。

汉书》，又名《前汉书》，中国古代历史著作。东汉班固所著，是中国第一部纪传体断代史。沿用《史记》的体例而略有变更，改“书”为“志”，改“列传”为“传”，改“本纪”为“纪”，无“世家”。全书包括纪十二篇，表八篇，志十篇，传七十篇，共一百篇，记载了上自西汉汉高祖元年（前206年），下至新朝地皇四年（23年），共229年历史。《汉书》语言庄严工整，多用排偶，遣辞造句典雅远奥，与《史记》平畅的口语化文字形成鲜明对照。中国纪史方式自《汉书》以后，都仿照其体例，纂修了纪传体的断代史。

自班彪起即以著《汉书》为己任，经过二十余年努力，班固完成了《汉书》的主要部分。汉和帝永元元年（89年），班固随从窦宪出击匈奴，参预谋议。后永元四年因事入狱并死在狱中。时《汉书》还有八表和《天文志》尚未写成，汉和帝命班昭（曹大家[1]）入东观藏书阁补作，马续协助班昭作《天文志》。故《汉书》前后历经四人之手完成，历时四十余年。在《汉书》注疏中，最著名者有唐颜师古注、清王先谦补注。

从手写 UI 到 AI 拼 UI：游戏界面开发方式的三次变化

  这几年做游戏开发，一个很明显的变化，就是 UI 的生产方式一直在变。十年前我在使用 cocos2d-x，那时候做界面基本就是“手搓”。按钮放哪、背景图怎么对齐、列表怎么排版、不同分辨率怎么适配，都要开发自己写代码处理。UI 不是不能做，而是成本很高，很多时间其实花在了重复劳动上。一个登录界面、一个背包界面，真正复杂的业务逻辑还没开始，光把节点摆好、响应接好，就已经消耗了不少精力。

  到了 2020 年以后，开始更多使用 Cocos Creator，情况就不一样了。Creator 把 UI 搭建从“写代码”变成了“可视化拼接”。节点层级、组件挂载、资源拖拽、布局调整，很多事情都可以直接在编辑器里完成。对开发来说，效率提升是非常明显的。以前要反复改坐标、改锚点、重新编译才能看到效果，现在在编辑器里拖一拖、点一点，立刻就能看到结果。某种意义上，这一代工具把 UI 开发门槛拉低了，也把很多机械性的工作从程序员手里拿走了。

  而现在，AI 又把这件事往前推了一步。它开始尝试“自动拼 UI”。也就是说，开发者不一定非要手动摆节点，也不一定非要自己一个控件一个控件去拖。你只要把需求描述清楚，AI 就有机会先给出一个初版界面，甚至把部分组件结构、样式层级、交互流程一并生成出来。从趋势上看，UI 的生产方式已经从“程序员自己实现”，走到了“工具辅助搭建”，再走到了“AI 参与生成”。

  不过我觉得，这里面最关键的一点不是 AI 会不会画界面，而是你有没有把足够准确的语料喂给它。这里的语料，不只是几句自然语言提示词，更重要的是和项目相关的 API、组件规范、命名规则、交互约束、页面结构，甚至是现有工程里的资源组织方式。AI 不是凭空理解你的项目，它生成出来的 UI 好不好，很大程度取决于你给它的信息是否完整。如果没有这些上下文，AI 能做的往往只是“看起来像那么回事”的 demo，而不是能真正接进项目里的 UI。

  所以我更倾向于把现在这个阶段理解成：AI 不是代替人来做 UI，而是把开发者的工作重心往前移了。以前我们主要在“实现”上花时间，现在更多是在“描述”和“约束”上花时间。你要先把接口定义清楚，把页面状态想明白，把组件边界整理好，再把这些内容交给 AI。谁的工程规范更清晰，谁的 API 设计更稳定，谁就更容易把 AI 用起来。

  从 cocos2d-x 到 Cocos Creator，再到今天 AI 开始拼 UI，我看到的不是某一种工具替代另一种工具，而是 UI 开发正在不断抽象、不断提效。手写代码解决的是“能不能做”，可视化编辑解决的是“能不能更快做”，AI 现在尝试解决的，则是“能不能先帮你做出第一版”。至于最后效果怎么样，还是取决于开发者对业务、工具和工程上下文的理解。

《中华古今注》：在名物之间保存古人的日常

  《中华古今注》这样的小书，最宜在夜深人静时翻读。它不厚，也不张扬，纸色已有旧意，版框方整，字口清楚，拿在手里先有一种温雅气。这样的书，不靠篇幅压人，也不以议论取胜，胜在一页页之间都藏着古人的生活痕迹。宫室城阙、车服器用、草木鸟兽，本是寻常名物，到了这类书里，却忽然都有了来历。

  今本《中华古今注》通常题作五代马缟撰。马缟在序中明言，晋人崔豹旧有《古今注》，见闻已广，只是尚有缺略，因此自己又“添其注，以释其义”。所以这书读来，很像一条文脉的续接，不是平地另起炉灶，而是在旧书基础上续作补苴，把前人留下的材料重新整饬，使其更便于后人寻绎。

  若从体例看，它近于古人的名物书。它并不专叙帝王功业，也不铺陈大段史事，而是从一个个名词入手，考其称谓，辨其沿革，间及制度、风俗与掌故。今天读到“宫”“阙”“城”“沟”这些字，往往只是纸上的概念；放回这类古书里，它们却都带着制度背景和生活气息。古代世界的轮廓，正是由这些细小处慢慢显现出来的。

  此书最可喜处，也恰在一个“杂”字。它所收甚博，却并不芜乱；篇幅虽短，却常有余味。往往只寥寥数语，便把一个名称的来源、一件器物的用途、一个旧说的转变说得大略分明。读者若把它当作闲书翻阅，自有兴趣；若把它当作入门，也能从中窥见传统名物学的门径。许多诗文中熟而不解的词，回头来查这类书，常会忽然明白几分。

  当然，《中华古今注》并非毫无疑义。后世目录家和考据家对《古今注》系统的真伪、增补、依托问题一直有讨论，认为其间并非一人一时之旧，而有层累附益之迹。正因如此，读它不必句句奉为定论。把它看成知识流转的痕迹，反而更有意味。抄录、转述、增补、重编，许多古书正是在这样的过程中慢慢定形；这本小书，也把那种传统学术生成的样子保存了下来。

  我向来偏爱这种小部头。它没有大书那样森严，却更近于案头清供。随手展开几叶，眼前浮出的也不只是几个术语，而是门阙、车辐、沟渠、衣冠、草木、禽兽所共同织成的一片古人日用世界。若从藏书的眼光看，这类书尤其耐人。它们未必显赫，却最能见时代纹理；未必堂皇，却常在细微处保存真气。

  这册《中华古今注》放在架上，也许并不最惹眼；但真抽出来细读，往往能生出绵长兴味。它提醒人，历史并不只在纪传、编年和大事记中，也藏在一个名称的解释、一件器物的辨析、一种习俗的追述里。能把这些细部保留下来的人，其实也替后来读书人留住了一部分古人的日常。藏书之乐，有时正是在这里。
