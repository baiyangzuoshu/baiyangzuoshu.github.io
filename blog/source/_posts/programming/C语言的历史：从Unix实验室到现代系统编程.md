&emsp;&emsp;C 语言是一门很特别的语言。它不算年轻，也不追求语法华丽，但直到今天，操作系统、编译器、数据库、嵌入式设备、运行时和大量底层库，仍然离不开它。理解 C 语言的历史，其实也是理解现代软件工业的一条主线：计算机从实验室走向工程化，软件从机器相关走向可移植，程序员从直接操纵硬件走向构建大型系统。

### 由来：从 CPL、BCPL 到 B

&emsp;&emsp;C 语言不是凭空出现的。它的血脉可以追溯到 1960 年代的 CPL 和 BCPL。CPL 试图成为一种强大的通用语言，但设计复杂；BCPL 则由 Martin Richards 在 1960 年代后期设计，目标更朴素：简单、紧凑、适合系统编程。

&emsp;&emsp;后来，Ken Thompson 在贝尔实验室基于 BCPL 设计了 B 语言。B 语言已经有了很多后来 C 的影子，但它主要面向早期机器，类型系统很弱。那个时代的硬件资源很紧张，语言必须足够小、足够贴近机器，否则根本跑不起来。

&emsp;&emsp;这就是 C 诞生前的背景：程序员需要一种比汇编高级、但又不像大型语言那样笨重的工具。它要能写操作系统，要能直接处理内存和机器字，又要能让代码在不同机器之间迁移。

### 诞生：Unix 需要一门新语言

&emsp;&emsp;C 语言真正诞生于贝尔实验室的 Unix 项目。1969 年前后，Ken Thompson、Dennis Ritchie 等人开始开发 Unix。早期 Unix 很多部分用汇编写，这在性能上直接，但在可维护性和移植性上都很困难。

&emsp;&emsp;Dennis Ritchie 在 B 语言基础上继续改进，加入更明确的类型系统、结构体、指针等能力，逐渐形成了 C 语言。大约到 1972 年，C 已经开始被用于 Unix 开发；到 1973 年，Unix 内核的重要部分被 C 重写。这一步非常关键，因为它证明了高级语言不仅能写应用，也能写操作系统。

&emsp;&emsp;C 的成功不只是语法成功，而是和 Unix 绑定在了一起。Unix 需要 C 来获得可移植性，C 也借 Unix 获得真实系统的锤炼。二者互相成就，最终影响了后来的操作系统、网络、编译器和整个开源软件世界。

### 发展：K&R C 与标准化

&emsp;&emsp;1978 年，Brian Kernighan 和 Dennis Ritchie 出版《The C Programming Language》。这本书后来被称为 K&R，它不仅是 C 语言教材，也在很长时间里承担了事实标准的角色。很多程序员就是通过这本书认识 C 的。

&emsp;&emsp;随着 Unix 传播，C 也快速扩散到高校、研究机构和商业公司。问题随之出现：不同编译器对 C 的实现并不完全一致。如果没有统一标准，代码可移植性就会受到影响。

&emsp;&emsp;于是标准化开始推进。1989 年，ANSI 发布 C 标准，通常称为 ANSI C 或 C89；1990 年，ISO 也发布对应标准，也就是 C90。这个阶段的意义在于，C 从贝尔实验室和 Unix 社区里的实用语言，变成了工业界共同遵守的标准语言。

&emsp;&emsp;之后 C 继续演进。C99 引入了很多现代化能力，比如 `//` 注释、变长数组、`inline`、更完整的整数类型等；C11 引入线程、原子操作和更明确的内存模型；C17 主要是修正和维护；目前最新正式发布的 ISO 标准是 ISO/IEC 9899:2024，通常称为 C23。它继续清理历史包袱，也补充了一些更适合现代开发的语言和库特性。

### 影响：现代语言的共同祖先

&emsp;&emsp;C 的影响远远超出了它本身。C++ 直接从 C 扩展而来，Objective-C、Java、C#、JavaScript、Go、Rust 等语言，都能看到 C 风格语法的影子。大括号、分号、表达式、控制流、函数调用方式，很多都成为后来程序语言的共同习惯。

&emsp;&emsp;更重要的是，C 建立了一套系统编程的思维方式。它让程序员清楚地看到内存、地址、指针、栈、堆、结构体和二进制布局。高级语言可以隐藏这些细节，但很多底层问题最终还是会回到这些概念上。

&emsp;&emsp;这也是为什么学习 C 仍然有价值。哪怕以后主要写 Java、Go、Rust 或 Python，理解 C 都能帮助你理解操作系统、网络、编译、性能和内存问题。C 像是一层底座，不总是在台前，但一直在支撑上层软件。

### 直到现在：仍在核心位置

&emsp;&emsp;今天，C 语言已经五十多年历史了。它当然有明显缺点：手动内存管理容易出错，指针带来安全风险，未定义行为让调试变复杂，大型工程也需要额外规范约束。很多安全问题，确实和 C/C++ 这类底层语言的内存模型有关。

&emsp;&emsp;但 C 仍然没有被替代。原因很现实：它足够贴近硬件，编译器成熟，运行时负担小，ABI 稳定，生态庞大。Linux 内核、嵌入式系统、数据库核心、图形库、语言解释器、驱动程序，仍然大量使用 C。很多新语言即使自己更安全，也需要和 C ABI 交互，因为 C 已经成为系统软件世界的共同接口。

&emsp;&emsp;所以 C 语言的历史不是一门旧语言被淘汰的历史，而是一门基础语言不断被重新定位的历史。它从 Unix 实验室走来，经历 K&R、ANSI、ISO 标准化，再走到 C23。今天它不再是所有场景的最佳选择，但在需要控制性能、内存、二进制接口和底层资源的地方，C 仍然是绕不开的语言。

&emsp;&emsp;理解 C，也是在理解软件和机器之间那层最直接的关系。
